## Házi feladat — Verem és Sor kiegészítés

### Feladat

- Valósítson meg egy generikus sor és verem adatszerkezetet, amelynek belsejében az elemek tárolásához láncolt listát használjon.
- Az adatszerkezeteken valósítsa meg a bejáró tervezési mintát.
- Az adatszerkezeteken valósítson meg operátor túlterhelést, `+` jel segítségével legyen lehetőség elemet berakni; `-` jel segítségével pedig elemet kivenni.

#### Megközelítés A

```csharp
class MyLinkedList<T>
{
    // láncolt listát alkotó logika...
}

class MyStack<T> {
    private MyLinkedList<T> items = new();
    // láncolt lista fel van használva, mint adatszerkezet (korábban tömb volt)
}
```

#### Megközelítés B

```csharp
class MyStack<T> {
    // maga a láncolt lista logika által van megírva a verem teljes belső működése
}
```

#### Pro és kontra

A verzió:

- Single Responsibility: a lista a lista logikáért felel, a verem a LIFO szabályért.
- Újrafelhasználhatóság: ugyanazt a listát használhatja a verem és a sor is (és akár később más struktúra).
- Karbantarthatóság / tesztelhetőség: külön teszteled a listát és külön a verem viselkedést.
- Kevesebb kódduplikáció: Queue/Stack esetén ugyanazok a műveletek ismétlődnének B-ben.
- Tisztább API: a verem publikus felülete marad Push/Pop/Peek, nem keveredik bele a lista belső működése.

B verzió:

- Direkt oktatási cél: "a verem belül láncolt listával működik" és nem akarsz plusz osztályt,
- Minimális implementáció kell (kevesebb fájl, kevesebb absztrakció),
- A lista "általános" implementációja túl sokat tudna / túl bonyolult lenne a feladathoz.
